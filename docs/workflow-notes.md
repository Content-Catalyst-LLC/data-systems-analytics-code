# Workflow Notes

The repository demonstrates a local-first analytical workflow:

1. Create a SQLite database from SQL schema and seed data.
2. Run SQL views and data-quality checks.
3. Use Python for ETL, profiling, feature generation, and exports.
4. Use R for statistical summaries and publication-oriented outputs.
5. Use Julia for numerical/time-series calculations.
6. Use Rust for fast validation of CSV files.
7. Use Go for streaming event simulation.
8. Use TypeScript for dashboard scaffolding.
9. Store outputs in `outputs/`.
10. Keep lineage and data contracts in `configs/`.
