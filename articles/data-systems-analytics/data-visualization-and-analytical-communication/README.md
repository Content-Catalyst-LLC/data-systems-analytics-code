# Data Visualization and Analytical Communication

This companion code models visualization and analytical communication as evidence-design infrastructure: chart choice, encoding quality, uncertainty placement, annotation, accessibility, audience fit, and review discipline.

The examples use visualization inventories, chart assessments, encoding checks, uncertainty elements, annotation records, accessibility checks, evidence links, audience contexts, and review checkpoints to show how teams can evaluate visual integrity, chart fit, perceptual clarity, uncertainty communication, accessibility, and responsible analytical framing.

## Included languages and examples

- `python/` — visualization integrity scorecard with manifest
- `r/` — chart, encoding, uncertainty, annotation, accessibility, and review summaries
- `julia/` — visualization readiness scoring
- `sql/` — schema and visualization-governance queries
- `go/` — visualization registry contract validation utility
- `rust/` — chart type and status inventory summarizer
- `c` — deterministic FNV-1a fingerprinting for visualization registries
- `cpp` — chart-to-evidence adjacency example
- `typescript/` — typed visualization, encoding, uncertainty, and accessibility contracts
- `quarto/` — reproducible visualization communication report skeleton
- `web/` — minimal static visualization communication mockup
- `terraform/` — non-deploying placeholder for visualization platform capabilities
- `bash/` — orchestration wrapper
- `docs/` — visualization design canvas, chart integrity checklist, accessibility checklist
- `ci/` — GitHub Actions smoke-test workflow

## Run locally

```bash
bash bash/run_all.sh
```

Optional language examples run only when the matching toolchain is installed.

## Core idea

Visualization makes evidence perceptible. Analytical communication makes it interpretable, contextual, accessible, and trustworthy.

```text
analytical question → chart choice → visual encoding
        → annotation + uncertainty + context
        → accessibility + review + versioned output
        → clearer reasoning and responsible interpretation
```

A mature visualization workflow does not only make charts. It preserves the relationship among question, comparison, evidence, uncertainty, design choice, and audience.
