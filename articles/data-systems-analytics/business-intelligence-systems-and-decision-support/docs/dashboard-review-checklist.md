# Dashboard review checklist

A high-trust dashboard should support a decision, not merely display information.

- [ ] The dashboard has a named owner.
- [ ] The primary decision context is clear.
- [ ] Certified metrics are distinguished from exploratory metrics.
- [ ] Freshness and quality status are visible.
- [ ] Users can inspect definitions or lineage.
- [ ] Thresholds or escalation pathways are documented.
- [ ] The interface avoids unnecessary visual clutter.
- [ ] Uncertainty or data limitations are visible where relevant.
- [ ] The dashboard has a lifecycle status.
- [ ] Deprecated dashboards have replacement guidance.
