# Relational Databases and SQL Systems

This companion code models relational databases and SQL systems as structured-data governance infrastructure: relations, tuples, attributes, keys, constraints, normalization, indexes, query workloads, transaction evidence, access controls, integrity incidents, and SQL-readiness scoring.

## Run locally

```bash
bash bash/run_all.sh
```

Optional language examples run only when the matching toolchain is installed.

## Core idea

Relational systems make structured institutional state explicit.

```text
domain entities → relations / tables
        → primary keys, foreign keys, constraints, domains
        → normalized schemas, indexes, transactions, access controls
        → declarative SQL queries, governed reporting, and readiness review
```

A mature relational system does not only store rows. It expresses identity, dependency, valid state, controlled change, transactional trust, query patterns, and auditable governance.
