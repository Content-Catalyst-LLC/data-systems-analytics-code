# Semantic governance checklist

- [ ] Certified metrics are separated from exploratory metrics.
- [ ] Multiple valid definitions are named and governed.
- [ ] Metric definitions include grain and time logic.
- [ ] Dimensions and entities are documented.
- [ ] Local dashboard calculations are reviewed for drift.
- [ ] Semantic definitions connect to lineage and tests.
- [ ] Access rules are applied where metrics expose sensitive information.
- [ ] Semantic changes have impact analysis.
- [ ] Deprecated metrics have replacement guidance.
- [ ] Adoption is monitored across consumption surfaces.
