# Analytics model review checklist

- [ ] Model grain is explicit.
- [ ] Owner and domain are identified.
- [ ] Business purpose is documented.
- [ ] Source dependencies are visible.
- [ ] Downstream dependencies are visible.
- [ ] Tests cover uniqueness, nulls, accepted values, reconciliation, or freshness where relevant.
- [ ] Critical dimensions and entities are documented.
- [ ] Metric definitions are not hidden in dashboard-only logic.
- [ ] Lifecycle status is current.
- [ ] Changes are versioned and communicated.
