# Observability checklist

- [ ] Freshness expectations are defined.
- [ ] Volume ranges or baselines are defined.
- [ ] Required schema fields are monitored.
- [ ] Distribution shifts are monitored for critical fields.
- [ ] Quality checks are tied to dataset owners.
- [ ] Alerts are prioritized by criticality and downstream impact.
- [ ] Lineage connects incidents to affected consumers.
- [ ] Incidents track acknowledgement and resolution.
- [ ] Recurring issues are reviewed.
- [ ] Semantic drift is reviewed with data stewards.
