# Data quality incident review checklist

- [ ] Incident severity is assigned.
- [ ] Affected dataset or product is identified.
- [ ] Upstream source and lineage impact are reviewed.
- [ ] Downstream dashboards, models, APIs, and reports are identified.
- [ ] Consumers are notified when decision-critical assets are affected.
- [ ] Root cause is categorized.
- [ ] Remediation owner is assigned.
- [ ] Temporary exceptions are documented.
- [ ] Recurrence risk is assessed.
- [ ] Prevention or monitoring improvements are recorded.
