# Match review checklist

- [ ] Candidate records refer to the same entity under the relevant use case.
- [ ] Required identifiers were compared.
- [ ] Attribute similarity was reviewed.
- [ ] Address, location, or jurisdiction differences were assessed.
- [ ] Legal and operational views were distinguished.
- [ ] False merge and missed match consequences were considered.
- [ ] Match confidence was separated from governance action.
- [ ] Steward decision was recorded.
- [ ] Effective date and historical state were preserved.
- [ ] Downstream impact was checked through lineage.
