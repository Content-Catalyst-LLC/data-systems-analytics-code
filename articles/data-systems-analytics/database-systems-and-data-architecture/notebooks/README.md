# Notebook guidance

Use notebooks to inspect database architecture evidence, not to hide official governance logic.

Recommended pattern:

1. Run `python/database_architecture_scorecard.py`.
2. Load `outputs/database_architecture_manifest_python.json`.
3. Inspect system readiness, asset scores, workload fit, lineage edge scores, risk scores, and estate summary.
4. Compare operational, analytical, streaming, metadata, archive, semantic, and ML data systems.
5. Preserve unresolved governance, recovery, and lineage risks before treating the estate as certified.
