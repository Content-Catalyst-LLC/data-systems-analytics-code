# Database systems and data architecture checklist

- [ ] Every critical system has an owner.
- [ ] Every major data asset has a grain, key, classification, and lifecycle status.
- [ ] Operational, analytical, streaming, metadata, and archive stores are distinguished.
- [ ] Workloads are mapped to systems and requirements.
- [ ] Metadata and lineage coverage are measured.
- [ ] Access controls are approved and reviewed.
- [ ] Quality gates protect promoted outputs.
- [ ] Backup and recovery plans meet RPO/RTO expectations.
- [ ] Integration contracts are documented.
- [ ] Architecture risks have owners and statuses.
