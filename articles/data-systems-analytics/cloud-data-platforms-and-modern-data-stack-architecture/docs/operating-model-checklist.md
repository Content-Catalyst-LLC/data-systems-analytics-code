# Operating model checklist

Cloud data architecture is an operating model, not only a diagram.

- [ ] Platform ownership is defined.
- [ ] Domain ownership is defined for important data products.
- [ ] Ingestion responsibilities are explicit.
- [ ] Transformation code is versioned and reviewed.
- [ ] Semantic definitions have stewards.
- [ ] Data products have lifecycle owners.
- [ ] Quality gates exist before promotion.
- [ ] Pipeline failures have alerting and escalation.
- [ ] Cost ownership is visible by workload or domain.
- [ ] Access policies are documented and audited.
- [ ] Metadata coverage is measured.
- [ ] Lineage coverage is measured.
- [ ] Retention policies exist for raw, curated, and published assets.
