# Reproducibility checklist

Use this checklist before treating an analytical result as durable or authoritative.

- [ ] The input dataset or snapshot is identifiable.
- [ ] The input fingerprint is recorded.
- [ ] The transformation code is in version control.
- [ ] The workflow can be run from a documented command.
- [ ] Runtime dependencies are declared.
- [ ] Parameters are captured.
- [ ] Output artifacts are written to a controlled location.
- [ ] Output fingerprints are recorded.
- [ ] Lineage connects output artifacts to input artifacts.
- [ ] The result has a run identifier.
- [ ] The run can be compared against a prior run.
- [ ] Review, validation, or quality checks are recorded.
