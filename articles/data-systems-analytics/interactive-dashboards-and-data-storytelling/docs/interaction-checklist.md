# Interaction checklist

- [ ] Dashboard mode is explicit: monitoring, exploration, or guided story.
- [ ] Default state is visible.
- [ ] Active filters are visible.
- [ ] Reset behavior is obvious.
- [ ] Filters match real user tasks.
- [ ] Drill-downs preserve context.
- [ ] Linked views deepen rather than distract.
- [ ] Tooltips supplement rather than hide critical meaning.
- [ ] Interactions are testable with common user tasks.
- [ ] The dashboard remains useful without excessive mode switching.
