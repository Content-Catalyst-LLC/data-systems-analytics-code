# Interactive Dashboards and Data Storytelling

This companion code models dashboards and data stories as governed analytical interfaces: monitoring surfaces, exploratory tools, narrative sequences, accessibility-sensitive views, and evidence-bearing decision products.

The examples use dashboard inventories, KPI definitions, filter controls, linked views, story points, annotations, visual elements, interaction events, accessibility checks, governance checks, and user tasks to show how teams can evaluate dashboard clarity, interaction burden, story coherence, KPI context, accessibility, evidence traceability, and responsible interface design.

## Included languages and examples

- `python/` — dashboard and story integrity scorecard with manifest
- `r/` — dashboard, KPI, filter, story, annotation, accessibility, and governance summaries
- `julia/` — dashboard readiness scoring
- `sql/` — schema and dashboard/storytelling governance queries
- `go/` — dashboard registry contract validation utility
- `rust/` — dashboard type and status inventory summarizer
- `c/` — deterministic FNV-1a fingerprinting for dashboard registries
- `cpp/` — story and dashboard evidence adjacency example
- `typescript/` — typed dashboard, KPI, filter, story, annotation, and accessibility contracts
- `quarto/` — reproducible dashboard/storytelling report skeleton
- `web/` — minimal static dashboard mockup
- `terraform/` — non-deploying placeholder for dashboard platform capabilities
- `bash/` — orchestration wrapper
- `docs/` — dashboard design canvas, interaction checklist, storytelling checklist
- `ci/` — GitHub Actions smoke-test workflow

## Run locally

```bash
bash bash/run_all.sh
```

Optional language examples run only when the matching toolchain is installed.

## Core idea

Dashboards support monitoring and exploration. Data stories support guided interpretation.

```text
analytical question → KPIs + context + views + filters
        → linked interaction + annotation + story sequence
        → accessibility + governance + review controls
        → reusable decision support and institutional memory
```

A mature dashboard or data story is not merely interactive. It makes the interface state, evidence, uncertainty, definitions, and intended use visible enough to support responsible interpretation.
