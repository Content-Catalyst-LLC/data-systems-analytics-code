# Mapping review checklist

A mapping should not be promoted until meaning and governance are clear.

- [ ] Source and target fields are documented.
- [ ] Transformation logic is explicit.
- [ ] Semantic risk is assessed.
- [ ] Identifier scope is understood.
- [ ] Controlled vocabulary translation is reviewed.
- [ ] Timing conventions are documented.
- [ ] Null and exception rules are defined.
- [ ] Mapping owner is assigned.
- [ ] Versioning and deprecation process exists.
- [ ] Downstream dependencies are visible.
