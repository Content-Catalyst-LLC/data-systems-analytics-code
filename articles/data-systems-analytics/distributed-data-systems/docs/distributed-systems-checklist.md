# Distributed data systems checklist

- [ ] Partition strategy is documented.
- [ ] Replication factor and placement policy are explicit.
- [ ] Read and write quorum rules are documented.
- [ ] Consistency model is stated per workload.
- [ ] Leader/follower or consensus roles are visible.
- [ ] Replica lag is monitored.
- [ ] Conflict-resolution rules are documented.
- [ ] Failover drills are tested and recorded.
- [ ] Operation logs preserve quorum and latency evidence.
- [ ] Repair, rebalancing, and recovery workflows are auditable.
