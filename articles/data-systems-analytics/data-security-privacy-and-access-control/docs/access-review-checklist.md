# Access review checklist

- [ ] Asset classification is current.
- [ ] Access is tied to a legitimate purpose.
- [ ] Permissions follow least privilege.
- [ ] Default access is deny, not allow.
- [ ] Temporary exceptions have expiration dates.
- [ ] Service accounts have named owners.
- [ ] Sensitive access is logged.
- [ ] Entitlements are reviewed on schedule.
- [ ] Stale groups and inherited roles are removed.
- [ ] Masking, tokenization, or row/column controls are applied where needed.
