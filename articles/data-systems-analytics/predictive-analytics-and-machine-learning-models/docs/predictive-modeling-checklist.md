# Predictive modeling checklist

- [ ] Prediction target is explicit.
- [ ] Prediction time and data availability are documented.
- [ ] Feature representation is reviewed.
- [ ] Train, validation, and test roles are distinct.
- [ ] Split strategy matches the problem structure.
- [ ] Preprocessing is fit only on training data or training folds.
- [ ] Metric family matches the task.
- [ ] Calibration is reviewed when probabilities drive decisions.
- [ ] Threshold policy is documented when scores trigger action.
- [ ] Rare-event and class-imbalance behavior is reviewed.
- [ ] Leakage and distribution-shift risks are checked.
- [ ] Monitoring windows and escalation rules are defined.
