# Warehouse, lake, and lakehouse checklist

- [ ] Raw data retention policy is explicit.
- [ ] Lake zones distinguish raw, bronze, silver, gold, sandbox, and archive.
- [ ] Warehouse marts define facts, dimensions, grain, and conformed dimensions.
- [ ] Metadata coverage and lineage coverage are measured.
- [ ] Ownership and classification are assigned.
- [ ] Access policies are applied by zone and asset.
- [ ] Data-swamp risk is reviewed.
- [ ] Table formats and transactional features are documented.
- [ ] Cost, query latency, scan efficiency, and storage growth are monitored.
- [ ] Workloads are matched to warehouse, lake, or lakehouse architecture intentionally.
