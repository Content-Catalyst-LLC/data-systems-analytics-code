# Data quality checklist

- [ ] Raw records are preserved.
- [ ] Quality dimensions are defined.
- [ ] Rules have owners, thresholds, and severity.
- [ ] Null, format, domain, duplicate, and freshness checks are executed.
- [ ] Rejected or flagged records include reason codes.
- [ ] Cleaning actions are logged.
- [ ] Duplicate resolution and survivorship rules are documented.
- [ ] Root causes are assigned to process owners.
- [ ] Quality incidents have stewardship status.
- [ ] Quality metrics are monitored over time.
